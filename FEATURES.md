### 4. 管理员审核功能

#### 审核流程：
1. 新注册的校友提交后状态为"待审核"（pending）
2. 管理员在后台查看待审核列表
3. 管理员可以选择"通过"或"拒绝"
4. 只有"已通过"（approved）的校友才会公开显示

#### 权限控制：
- **总管理员**：可审核所有校友
- **学校管理员**：只能审核本校校友
- **班级管理员**：只能审核本班校友

#### API接口：
```
GET  /api/alumni/pending        # 获取待审核列表
POST /api/alumni/:id/approve    # 审核通过
POST /api/alumni/:id/reject     # 审核拒绝
```

## 📝 注意事项

1. **首次运行**：服务器会自动创建数据库和初始化数据
2. **重新初始化**：删除 `server/database.sqlite` 后重启服务器
3. **网络要求**：头像需要联网加载（Unsplash CDN）
4. **浏览器缓存**：如遇到问题，清除浏览器缓存
5. **生产环境**：请修改默认密码，配置HTTPS

## 🐛 常见问题

**Q: 游客可以看到校友详细信息吗？**
A: 不可以，只能看到列表，点击详情会提示登录。

**Q: 学校管理员能看到其他学校的数据吗？**
A: 不能，系统会自动过滤，只能看到本校数据。

**Q: 如何添加更多管理员？**
A: 使用总管理员账号登录，通过API或数据库直接添加。

**Q: 头像加载失败怎么办？**
A: 检查网络连接，Unsplash在国内可能需要代理。可以替换为本地头像。

**Q: 如何重置数据库？**
A: 删除 `server/database.sqlite` 文件，重启服务器会自动重建。
