## 技术转换对照

| Web版本 | 小程序版本 | 说明 |
|---------|-----------|------|
| HTML | WXML | 标签名变化（div→view, span→text等） |
| CSS | WXSS | 基本兼容，部分选择器限制 |
| JavaScript | JavaScript | ES6语法，API不同 |
| localStorage | wx.setStorageSync | 本地存储API |
| fetch | wx.request | 网络请求API |
| DOM操作 | 数据绑定 | 从命令式到声明式 |
| window对象 | getApp() | 全局对象访问 |
| document | Page/Component | 页面和组件 |

## 使用指南

### 开发环境
1. 安装微信开发者工具
2. 导入 `wechat-miniprogram` 目录
3. 配置后端API地址（app.js中的baseUrl）
4. 编译运行

### 后端要求
确保Java后端已启动并支持以下CORS配置：
```java
// 允许小程序域名访问
@CrossOrigin(origins = {"https://servicewechat.com"})
```

### 生产部署
1. 在微信公众平台注册小程序
2. 配置request域名（必须是HTTPS）
3. 修改app.js中的baseUrl为生产地址
4. 在开发者工具中上传代码
5. 提交审核
6. 发布上线

## 性能优化建议

1. **图片优化**
   - 使用压缩后的图片
   - 适当使用懒加载
   - 头像使用小尺寸

2. **数据缓存**
   - 合理使用本地缓存
   - 避免重复请求
   - 实现下拉刷新

3. **分包加载**
   - 管理后台可作为分包
   - 减少主包体积

4. **按需加载**
   - 使用wx:if代替hidden
   - 列表使用虚拟列表

## 注意事项

1. **域名白名单** - 上线前必须在小程序后台配置服务器域名
2. **HTTPS** - 生产环境必须使用HTTPS
3. **Token安全** - 注意Token的存储和刷新机制
4. **权限验证** - 前端权限控制仅用于UI展示，后端必须做权限验证
5. **数据校验** - 前后端都需要做数据校验
